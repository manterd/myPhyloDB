What is myPhyloDB?
  • A user-friendly web-interface for accessing and analyzing taxonomic data from multiple projects and/or sequencing runs.
  • A more detailed reference manual can be found here.
  • The goal of myPhyloDB is to allow for easy comparisons and statistical analysis of microbial (i.e., fungi or bacteria) taxonomic abundance across projects, environments, and management scenarios.
  • myPhyloDB accepts Mothur pre-analyzed data from any its supported sequencing platforms. In addition, raw 454 and Illumina/MiSeq files may be uploaded and analyzed using a user-customizable Mothur pipeline.

myPhyloDB's Unique Features
  • myPhyloDB stores sample information, taxonomic data and counts from multiple projects in a single SQL database,
    which is suitable for a wide variety of project types (e.g., soil, human-associated, human gut, air, water, microbial).
  • All of the necessary back-end programs (e.g., Mothur, R, Python) are embedded within myPhyloDB. No additional
    software, or compatibility issues, to worry about.
  • Adding/editing samples, metadata, and/or sequencing reads are all automated with myPhyloDB's graphical interface.
  • Experienced Mothur users may use their own scripts for customized sequence processing pipelines.
  • myPhyloDB can perform a variety of normalization procedures to ensure equal sampling efforts across all selected projects/samples.

Data Input

myPhyloDB currently requires the following data files for each project.
Example files can be downloaded using the following links:

    Example 1. Soil project with pre-analyzed Files
  • Project File: General project information.
  • Sample File: Meta data for each sample including Mimark compliant sample information, edaphic (soil) factors, and management options.
  • Shared File: Species (or operational taxonomic unit, OTU) table based on the mothur-format as described here.
  • Taxonomy File: Consensus taxonomy file as described here.

  • Sample File: Meta data for each sample including Mimark compliant sample information, edaphic (soil) factors, and management options.

  • Oligos File: File with barcode/primer and sample info as described here.
  • Mothur Batch File: Batch file for pre-processing as described here.

    • Example 3. Soil project with raw Illumina/MiSeq Files
  • Project File: General project information.

  • Reverse Fastq File: Example raw MiSeq reverse fastq file with barcodes/primers removed.
  • Mothur Batch File: Batch file for pre-processing as described here.

Analysis Options

Samples can be selected from any project currently stored in the user's database. Once selected, taxonomy alpha-diversity (e.g., relative abundance, species richness, or Shannon's diversity) can be compared graphically and statistically (ANOVA) for any meta variable contained in the database. Beta-diversity (e.g., Bray-Curtis distance) can also be graphically (PCoA, Principal Coordinates Analysis) and statistically (perMANOVA, betaDisper) compared for any meta variable and taxonomic level contained in the database.
