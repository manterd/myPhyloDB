What is myPhyloDB?

A user-friendly web-interface for accessing and analyzing taxonomic data from multiple projects and/or sequencing runs.
A more detailed reference manual can be found here.

The goal of myPhyloDB is to allow for easy comparisons and statistical analysis of microbial (i.e., fungi or bacteria) taxonomic abundance across projects, soil types, and management scenarios.

Data may be obtained from any sequencing platform; however, currently only mothur-formatted files are supported.


Database:

myPhyloDB stores sample information, taxonomic data and counts from multiple projects in a single SQL database.

Data Input:

myPhyloDB currently requires four data files for each stored project.
Example files can be downloaded using the following links:

  • Project File: General project information.
  • Sample File: Meta data for each sample including Mimark compliant sample information, edaphic (soil) factors, and management options.
  • Shared File: Species (or operational taxonomic unit, OTU) table based on the mothur-format as described here.
  • Taxonomy File: Consensus taxonomy file as described here.

Analysis Options:

Samples can be selected from any project currently stored in the user's database. Once selected, taxonomy alpha-diversity (e.g., raw counts, relative abundance, species richness, or Shannon's diversity) can be compared graphically and statistically (ANOVA) for any meta variable contained in the database. Beta-diversity (e.g., Bray-Curtis distance) can also be graphically (PCoA, Principal Coordinates Analysis) and statistically (AMOVA, HOMOVA) compared for any meta variable and taxonomic level contained in the database.
